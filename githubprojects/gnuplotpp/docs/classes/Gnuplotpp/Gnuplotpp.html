N/A Gnuplotpp(bool persist = true, size_t buffSize = 1 << 10)
Default constructor
Params
bool persist = true
Opens the pipe with --persist option If this value is set to true, the pipe with gnuplot will be open by calling 
gnuplotpp --persist
otherwise it will be openen with a simple: 
gnuplotpp
size_t buffSize = 1 << 10
The internal streambuf buffer size With this constructor, an internal GnuplotPipe will be created: this class is an ostream with a default streambuf that is a PipeStreambuf??, it holds and interal buffer that is written every time it is flushed into the pipe (for example by std::endl or every time it reaches buffSize).
Return Value
n/a n/a
This is the default costructor. It is called for example when you build a Gnuplotpp instance like this: 
// this will call the default constructor
Gnuplotpp  gp;
This constructor will create an instance with only one streambuffer: a pipe created calling (on terminal) gnuplot --persist.
The --persist option can be removed by setting persist=false.
The internal buffer size for the internal streambuf cna be controled with befSize but it might not be very usefull for the user.
N/A Gnuplotpp(std::ofstream&& oFileStream)
Instance to write on file
Params
std::ofstream&&oFileStream
Output file stream

The output file stream where the commands will be wrote.

The value is taken by rvalue reference (double reference) so that it is moved.

Return value
N/AN/A

This constructor builds a Gnuplotpp instance that writes all the output in the oFileStream file.
The Gnuplot program is not called and no plots will be created.

Example: 

{
    // This instance will only print the output on the "out.p" file
    Gnuplotpp gp(std::ofstream("out.p"));

    // example direct command:
    gp << "plot sin(x)" << std::endl;

    // here the file is closed
}

// you can execute the commands by writing on console:
std::system("gnuplot out.p");
// or by opening the command line, starting gnuplot and
// typing "load out.p"

N/A Gnuplotpp(std::shared_ptr<std::ostream> ostream)
Builds on a smart pointer to an ostream
Params
std::shared_ptr<std::ostream> ostream
Pointer to the stream The shared pointer to the stream where the object will put the output.
Return value
N/AN/A

This constructor creates an instance without calling the gnuplot program. The output is sent to the given ostream.

Example: 

// Create a smart pointer to a string stream
auto ss = std::make_shared<std::sstream>();

// create a 'Gnuplotpp' instance with output on 'ss'
Gnuplotpp gp(ss);

// example direct command:
gp << "plot sin(x)" << std::endl;

// print the output written is ss
std::endl << ss.str() << std::endl

void setTerminal(Terminal term = Terminal::None, std::optional<std::string> outFile = {}, std::optional<Vector2i> size = {});
Set a Gnuplot terminal type
Params
Terminal term = Terminal::None
The terminal type This is the gnuplot terminal type. The LC library makes available only some common terminal types. Additional terminal tyes for the current installation can be tested with: 
help terminal
and 
test terminal
Add the possibility to type other terminal names with a string
std::optional<std::string> outFile = {}
Output file name Some terminals require an output file, for example JPG, PDF, SVG, LATEX, ...
size, change to float or double
Returns
void N/A

Gnuplot can draw on different terminal types, for example, on windows the default terminal might be of type "windows" or "qt".
To sare images, for example, you might choose an appropriate terminal, for example JPEG and set an output file.

Example 
// Create a Gnuplotpp instance
Gnuplotpp gp;

// Set a terminal that allows us to save images.
// Here, for example, we are choosing a JPEG
// terminal that will the plot as an image.
// We also have to specify the output file.
gp.setTerminal(Gnuplotpp::Terminal::JPEG, "out.jpg");

// Plot something
gp << "plot sin(x) << std::endl;

// Image will be saved on scope exit...
